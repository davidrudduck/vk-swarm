```rust

// After: configurable with sensible default
```

- [x] Test light and dark mode - no contrast issues
- [x] Run clippy - no warnings
- [x] Run all tests - all pass (19 db tests, 25 remote tests)

### Implementation Details

**New composite index:**
```sql
CREATE INDEX IF NOT EXISTS idx_tasks_project_activity_at
ON tasks(project_id, activity_at DESC);
```

This index optimizes the common query pattern:
```sql
WHERE project_id = ? ORDER BY COALESCE(activity_at, created_at) DESC
```

The existing simple `idx_tasks_activity_at` index only helps with global sorting. This composite index covers the typical case of project-scoped task listings, which is the most common use case in the application.
