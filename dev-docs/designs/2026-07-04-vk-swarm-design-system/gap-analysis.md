# Gap analysis — vk-swarm-design-system

**Design spec:** `dev-docs/designs/2026-07-04-vk-swarm-design-system/design-spec.md`
**Compared against:** `frontend/` (node frontend) + `remote-frontend/` (hive frontend, post tasks 100-201).

## Summary

The design introduces a **`.vks-*` component-class vocabulary** (`.vks-btn`, `.vks-card`, `.vks-task`, `.vks-node`, `.vks-badge`, `.vks-input`, `.vks-switch`, `.vks-checkbox`, `.vks-tabs`, `.vks-select`, `.vks-loader`, `.vks-status`, `.vks-settings`, `.vks-alert`, `.vks-savebar`) plus refined texture utilities and a token set. The **node frontend already implements** the brand palette (`--vks-*` color tokens at `frontend/src/styles/index.css:64-76`), partial texture utilities (`.vks-ansi-dither`, `.vks-scanlines` at `frontend/src/styles/index.css:9-12`), and feature components (kanban, TaskCard, StatusBadge, NodeCard) — but built on **shadcn/ui primitives + Tailwind utility classes**, not the `.vks-*` component classes the design specifies. The **remote-frontend** has only the auth shell (ProfileProvider, oauthApi, AppRouter, slim NormalLayout/Navbar/BottomNav from tasks 100-106) and none of the design system.

This is overwhelmingly **(C) un-built scope**: the `.vks-*` component classes, refined tokens (typography scale, spacing, radius, motion, glows), remaining texture utilities, and the hive app UI kit (BoardView, Chrome, Panels/TaskDrawer) do not exist in either frontend.

## Routing decision

This is **C-class scope** (un-built design system + hive app UI kit). Per the skill's Step 3 decision rule, C-class requires the full WAI pipeline:

1. Create `dev-docs/workstreams/vk-swarm-design-system/README.md`
2. Author WAI spec at `docs/superpowers/specs/2026-07-04-vk-swarm-design-system.md`
3. Run `/wai:precheck → /wai:decompose`

The existing `vk-swarm-hive-ui` workstream (tasks 100-201 done, 202-308 superseded) should be closed/superseded — its phase 1 (auth shell, tasks 100-106) is reusable infrastructure for the new design-system workstream's hive app shell.
