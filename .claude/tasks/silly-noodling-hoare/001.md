# Task: Sort Executor Type Dropdown in AgentSettings

## Goal
Add alphabetical sorting to the executor type dropdown on the Settings → Agents page.

## Context
The Settings → Agents page (`AgentSettings.tsx`) has two dropdowns:
1. **Executor Type** dropdown - shows agent types like CLAUDE_CODE, CODEX, GEMINI, etc.
2. **Configuration** dropdown - shows variant configurations for the selected executor

Currently these dropdowns display items in object key order (unordered). Other parts of the app (task forms) already sort these alphabetically. This task adds `.sort()` to the executor type dropdown.

## Steps
1. Open `frontend/src/pages/settings/AgentSettings.tsx`
2. Find line 429 where the executor type dropdown items are mapped
3. Add `.sort()` after `Object.keys(localParsedProfiles.executors)` and before `.map()`

## Files to Modify
- `frontend/src/pages/settings/AgentSettings.tsx` - line 429: add `.sort()` to executor type mapping

## Code

Find this code block (around line 428-435):
```typescript
                    <SelectContent>
                      {Object.keys(localParsedProfiles.executors).map(
                        (type) => (
                          <SelectItem key={type} value={type}>
                            {type}
                          </SelectItem>
                        )
                      )}
                    </SelectContent>
```

Change it to (add `.sort()` after `Object.keys(...)`):
```typescript
                    <SelectContent>
                      {Object.keys(localParsedProfiles.executors).sort().map(
                        (type) => (
                          <SelectItem key={type} value={type}>
                            {type}
                          </SelectItem>
                        )
                      )}
                    </SelectContent>
```

The only change is on line 429:
- FROM: `{Object.keys(localParsedProfiles.executors).map(`
- TO: `{Object.keys(localParsedProfiles.executors).sort().map(`
