## What are task tags?

Task tags (also known as templates) are reusable text snippets that you can quickly insert into task descriptions. There are two ways to insert them:

1. **@mention** - Type `@` followed by the tag name in any description field
2. **Insert Template button** - Click the template icon in the task form toolbar

When you select a template, its content is automatically inserted at your cursor position.

<Tip>
Task tags use snake_case naming (no spaces allowed). For example: `bug_report`, `feature_request`, or `code_review_checklist`.
</Tip>

## Using task tags

Insert task tags into task descriptions and follow-up messages using @mention autocomplete.

<Steps>
<Step title="Trigger autocomplete">
  When creating or editing a task description, type `@` to trigger the autocomplete dropdown.

  <Frame>
  <img src="/images/screenshot-task-tag-autocomplete.png" alt="Autocomplete dropdown showing available tags after typing @ symbol" />
  </Frame>
</Step>

<Step title="Search and select">
  Continue typing to filter tags by name, then:
  - Click on a tag to select it
  - Use arrow keys to navigate and press Enter to select
  - Press Escape to close the dropdown

  <Check>
  The tag's content is automatically inserted at your cursor position, replacing the @query.
  </Check>
</Step>
</Steps>

## Using the Insert Template button

The task form includes an **Insert Template** button in the toolbar (next to the file search and image upload buttons). This provides a visual way to browse and insert templates.

<Steps>
<Step title="Open the template picker">
  Click the **Insert Template** button (document icon) in the task form toolbar. A template picker dialog will open.
</Step>

<Step title="Browse or search templates">
  The picker shows both built-in templates and your custom task tags. Use the search field to filter templates by name or description.
</Step>

<Step title="Select a template">
  Click on any template to insert its content at your cursor position in the description field. The picker closes automatically after selection.
</Step>
</Steps>

<Info>
On mobile devices, the template picker appears as a bottom sheet for easier touch interaction.
</Info>

## Built-in templates

Vibe Kanban includes four built-in templates that are always available:

| Template | Description |
|----------|-------------|
| **Bug Report** | Structured format for reporting bugs with steps to reproduce, expected vs actual behaviour, and environment details |
| **Feature Request** | Template for proposing new features with problem statement, proposed solution, and alternatives |
| **Code Review Checklist** | Comprehensive checklist covering functionality, code quality, testing, security, and documentation |
| **Quick Task** | Minimal template with goal, acceptance criteria, and notes sections |

<Tip>
Built-in templates appear alongside your custom task tags when using the Insert Template button. You cannot modify or delete built-in templates, but you can create custom tags with similar names to override them for @mention autocomplete.
</Tip>

<Tip>
